# AGENTS - Frontend App

## Mission Profile
AI Learning Support UI built with Flask for nursing education. Features include Onboarding, Course Learning, Quizzes, and an interactive audio-visual KI-Lehrer (AI Teacher) with realistic avatar and speech recognition.

## Current Implementation Status

### Architecture
- **Backend:** Flask app (`app.py`), SQLAlchemy ORM (`models.py`), business logic (`services.py`)
- **Frontend:** Server-rendered templates (Jinja2) with vanilla JavaScript
- **Database:** SQLAlchemy 2.0.41 with models for Users, Courses, Modules, Quizzes, Enrollments, Spaced Repetition
- **External Microservices:**
  - **RAG API** (port 8000): Document ingestion, indexing, and storage
  - **LLM API** (port 8001): AI generation for chat, quizzes, and presentations

### Avatar & Visual Features
- **KI-Lehrer Avatar:** TalkingHead.js library with realistic 3D avatar (brunette.glb model from CDN)
  - Phoneme-based lip sync with audio from `/api/tts-proxy`
  - Mood/expression support (neutral, happy, sad, etc.)
  - Fallback: 🎓 emoji if avatar fails to load
- **Slide Generation:** Auto-generated by external LLM API during `/generate_presentation`
  - JSON format: Array of slides `[{ "title": "...", "bullets": [...], "images": [...] }, ...]`
  - **Schema Update Fix (NEW):** Frontend parsing now consumes nested `slides[]` payload directly (including per-slide `spoken_text`) instead of relying on deprecated top-level keys, resolving `TypeError` crashes in KI-Lehrer.
   - **Carousel Feature (REFACTORED):** KI-Lehrer uses event-driven async/await for perfect audio-visual synchronization
    - Backend returns `slides[]` array; frontend renders as carousel with only first slide visible initially
      - Event-driven sync: async `playPresentation()` loop triggers slide speech, then waits for debounced avatar silence before advancing
      - Audio wait guarantee: a Debounced Silence Detector polls TalkingHead speaking state (`isSpeaking` + queue/state guards) every 500ms and requires 1.5s sustained silence, preventing premature transitions when TTS is chunked into multiple parts
    - This prevents instant skipping to the last slide when speech backends do not return an awaitable completion signal
    - Each slide's audio completion triggers automatic advance to next slide; no timing estimation needed
    - Manual navigation (prev/next buttons or dot selection) stops the auto-advance timer
    - When user manually navigates, the carousel returns to normal slide-by-slide control
  - Integrates with KI-Lehrer visual UI and TalkingHead avatar lip-sync
  - Images: Returned from backend as relative paths; JavaScript forcefully prepends `http://localhost:8000` to prevent port 5000 404s
    - **Bug fix:** Double `/api/v1` deduplication added (`.replace('/api/v1/api/v1/', '/api/v1/')`) to clean up duplicated path segments when backend prepends full URL to already-qualified paths
  - Only the first image per slide is rendered (no duplicates); console logs each image URL attempted for debugging
  - KI-Lehrer auto-trigger on page load is disabled; backend calls start only after explicit user action (send/mic)
  - Slide layout uses side-by-side flex columns (text with bullets on left, image on right)
  - **Traceability button (NEW):** Slides now show a small bottom-right "📄 Seite X" button when `source_page` is present, so learners can see which original PDF page grounded the content
  - Responsive: Falls back to column layout (stacked) on screens narrower than 980px
- **Speech-to-Text (STT) Robustness (IMPROVED):**
  - Frontend guard: Empty audio blobs are detected (`blob.size === 0`) before sending to `/api/stt`, preventing unnecessary requests
  - Backend guard: `/api/stt` now returns graceful HTTP 200 response `{ "error": "no_audio", "text": "" }` instead of HTTP 400 crash when audio is empty

## Docker Networking Note
- Frontend runs inside Docker, so `localhost` inside the container refers to itself, not the host.
- `docker-compose.yml` now sets `extra_hosts: ["host.docker.internal:host-gateway"]` on the `frontend` service.
- Backend API defaults for container runtime are:
  - `RAG_API_BASE_URL=http://host.docker.internal:8000/api/v1`
  - `LLM_API_BASE_URL=http://host.docker.internal:8001`
- This allows the Dockerized Flask frontend to reach host-based RAG/LLM microservices reliably in local development.

## API Routes Added/Modified

### New Upload Routes (Teacher-Only)
- **`POST /api/upload`** - Upload PDF document
  - Forwards to RAG API: `POST /courses/{course_id}/documents`
  - Returns: `{ "doc_id": "...", "message": "..." }`
  
- **`GET /api/upload/ready/<course_id>/<doc_id>`** - Check document readiness
  - Polls RAG API: `GET /courses/{course_id}/documents/{doc_id}/ready`
  - Returns: `{ "ready": true/false, "indexed_chunks": int }`

### Modified AI Routes (Now Using External LLM API)
- **`POST /api/ai-professor`** - AI-Professor QA chat
  - Now calls: `POST http://localhost:8001/generate_answer`
  
- **`POST /api/ki-lehrer`** - KI-Lehrer multi-turn presentation
  - Now calls: `POST http://localhost:8001/generate_presentation`
  - Returns: `{ "response": "...", "slide_title": "...", "slide_points": [...], "images": [...] }`
  
- **`GET /api/quiz/generate/<module_id>`** - Quiz generation
  - Now calls: `POST http://localhost:8001/generate_quiz`

### Preserved Audio Routes (UNCHANGED)
- **`POST /api/tts-proxy`** - ElevenLabs/edge-tts (for avatar voice)
- **`POST /api/stt`** - OpenAI Whisper (for voice input)
